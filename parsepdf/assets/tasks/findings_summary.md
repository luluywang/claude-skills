# Findings Summary

Generate a structured summary of all tables and key quantitative findings from the assembled markdown document. This output is designed to help future agents quickly understand what data is available and what the paper finds.

## Input

- Assembled markdown: `output/[PAPER_NAME].md`
- Document structure: `work/[PAPER_NAME]/structure.json`

## Task

Read the full assembled markdown and produce a findings summary with two parts:

### Part 1: Table-by-Table Inventory

For each table in the document, write one entry with:
- Table number and title
- What the table shows (one sentence)
- Key columns / outcome variables
- Sample and specification (e.g., "OLS, N=10,432, state FE")
- 1-3 key coefficient estimates with significance stars, exactly as reported (e.g., "β = 0.042***, se = 0.008")

### Part 2: Key Quantitative Findings

List the 5-10 most important numerical results from the paper. For each:
- The finding in plain language
- The exact number(s) as reported (coefficient, elasticity, share, etc.)
- Where it appears (Table N, page N, or section name)

Focus on:
- Main causal estimates or treatment effects
- Magnitudes and economic significance
- Robustness results that confirm or bound the main finding
- Any decomposition percentages or share calculations

## Output Format

Write to `output/[PAPER_NAME]_findings.md`:

```markdown
# Findings Summary: [Paper Title]

*Generated by parsepdf. Intended for quick reference by future agents.*

---

## Tables

### Table 1: [Title]
- **Shows**: [One sentence describing what this table estimates]
- **Outcome**: [Dependent variable(s)]
- **Specification**: [Method, sample size, fixed effects]
- **Key estimates**:
  - [Variable]: [coefficient] ([significance]), se = [se]
  - [Variable]: [coefficient] ([significance]), se = [se]

### Table 2: [Title]
...

---

## Key Quantitative Findings

1. **[Finding label]**: [Plain-language description]. Exact value: [number with units]. Source: [Table N / Section name].

2. **[Finding label]**: ...

...

---

## Notes for Future Agents

- Tables are in `output/[PAPER_NAME].md` under the Results section
- Raw table JSON: `work/[PAPER_NAME]/tables/`
- QA report: `output/[PAPER_NAME]_qa.json`
```

## Notes

- Copy coefficient values exactly as they appear — do not round or reformat
- If a table has many columns, focus on the preferred specification (usually the last or most demanding column)
- If the paper has no tables (theory paper), note that and focus Part 2 on propositions or calibrated parameter values
- Skip appendix tables unless they contain the main robustness results
